Git & Terminal

Git det er et værktøj, der bliver brugt til versionsstyring. Det er et distribueret versionsstyringssystem. Det bruges under softwareudvikling, hvor det kan bruges til at spore ændringer i koden. Det er vigtig at kunne finde ud af at bruge git-kommandoer i terminalen, for at administrere versioner af projektet, også samarbejde med teamedlemmer, udviklingshold og spore ændringer af koden i projektet effektivt.

Et eksempel på hvordan man stage ændringer man har lavet i koden og commit med en commit message og push det til den branch man arbejder i.

Det er kommandoen til at stage ændringer af koden i filer git add -A
Den her kommando vil stage alle filer, hvor der har lavet ændringer. Ved at bruge -A, så vil den også stage ændringer på filer, som ligger i andre directory, mapper, hvis nu man har et projekt med mange mapper og undermapper hvor alle filer ikke bare ligger i en root mappe. Derimod kommandoen git add . vil kun stage ændringer på filer, i ens nuværende lokation, altså den mappe man er i, root folder.

Kommandoen til at commit stage filer med en commit message, det er git commit -m "message", når der commit så skal man skrive en besked, for hvilke ændringer man har lavet, så inden i "" double quote, så skal man skrive en besked der beskriver hvad man har lavet.

Kommandoen git push bruges efter man har commit ændringerne, hvor det så vil blive push til github så andre udvikler eller den man samarbejde kan hente de nye ændringer, som gøres ved kommandoen git pull

En liste af kommandoerne:
git add -A eller git add .
git commit -m "message"
git push
git pull
