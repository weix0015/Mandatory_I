Import / Export HTML

I HTML kan man import og export, og det er når man bruger JavaScript moduler.
<script type="module"></script>, det er i script tag. Det betyder, altså når vi importerer et JavaScript modul i HTML, så bruger vi <script type="module"></script>. Her fortæller vi browseren, at det javascript det er et javascript modul, det tillader os at dele vores javascript kode op i separate filer. Det gør det lettere at håndtere og vedligeholde vores kode.
Det gør også vi kan genbruge funktioner, variable og objekter. Her er eksempler på at dele det op i separate javascript filer. 

// addition.js
export function add(a, b) {
  return a + b;
}
// subtraction.js
export function subtract(a, b) {
  return a - b;
}
Her har vi 2 separate javascript filer, som vi kan genbruge i en anden fil. 
// app.js
import { add } from "./addition.js";
import { subtract } from "./subtraction.js";

console.log(add(5, 3)); // Output: 8
console.log(subtract(10, 4)); // Output: 6
her i eksemplet kan vi forbinde app.js module i vores html fil. <script src="app.js" type="module"></script> så har vi også isolation, så hvis vi ændre fx. addition.js, så vil det ikke påvirke de andre javascript filer. Det er også skalarbart, da vi kan tilføje nye funktionaliteter eller fjerne funktionaliteter ved at tilføje moduler efter behov. Det gør også vi undgår at have mange script tag, i en html fil.

Export i javascript, der kan vi gøre funktioner, variabler eller objekter tilgængelig for andre moduler eller filer. Hvor vi kan genbruge kode og holde det afskilt fra hinanden.
Der er 2 hovedtyper af export, der er named exports, hvor vi specificerer, hvilke funktioner, variabler eller objekter vi gerne vil exportere, hvor vi bruger export keyword før funktion eller før variablen / objektet.
Her er et eksempelt. 
export function greet(name) {
  return "Hello, " + name + "!";
}

export const message = "Welcome here!"
I dette eksempelt, så gør vi andre filer og moduler har adgang til vores greet function og message variable. Der findes også Default export, når vi export default fra et modul som standard, så når en anden fil importere, det så kan de kalde instansen hvilket som helst andet navn, som de har behov for, i stedet for at bruge det samme navn.
For eksempel 
export default function greet(name) {
  return "Hello, " + name + "!";
}

export default class User {
  constructor(name) {
    this.name = name;
  }  
}
Så hvis de skal bruge de funktioner og variable eller objekter, fra en anden fil, et andet JavaScript-modul, så skal de importeres, det gør vi ved at bruge import keyword. Hvor vi angiver hvilken path den fil ligger i og specificerer de elementer, som vi gerne vil importere inde i { }.
Det er et eksempelt, hvis vi skal importere named exports 
import { greet } from "./greeting.js";
import { message } from "./messages.js"
Her importere vi funktionen greet fra filen greeting.js og variablen messsage fra filen message.js.
Hvis vi gerne vil importere default import, som er blevet eksporteret som standard fra et modul. 
import greet from "./greeting.js";

import User from "./User.js";
Man kan også importere og give det et alias, altså give funktioner, variabler eller objekter et andet navn. 
import { greet as sayHello } from "./greeting.js";
Forskellen mellem default export og export, det er at, hvis man bruger default export, så man kan kun eksportere en ting fra et modul som standard. Det betyder altså, når du importerer en default export i et andet modul, så kan du give det hvilket som helst navn, og det vil også sige, at hvis du bruger export default, så kan du kun eksportere en default export, som betyder kun 1 funktioner, eller variable, klasse eller objekt kan blive eksporteret som standard fra en fil.
Du kan selvfølgelig have 1 default export og en named export, så du både kan importere det, hvis man skal bruge begge dele.
I named export, så kan man eksportere flere funktioner, variabler eller objekter fra et modul. Hvis du importerer named exports i et andet modul, så skal du bruge de præcise navne, som de blev export med. I named export så skal man også bruge { }, hvor man angiver hvilke elementer man vil importere i filen.
