Import / Export Node.JS

Der er 2 primære måder at importere og eksportere moduler i Node.js på.
Der er CommonJS og ECMAScript moduler (ES-moduler), det er også kendt som type module eller type=module.

CommonJS det er kendt som den oprindelige modulstandard i Node.js. Når man arbejder med CommonJS, så bruger man require(), og det er til at importere moduler og man bruger module.exports eller exports objektet, hvis man skal eksportere moduler. Her er et eksempelt. 

const express = require("express"); // import express in CommonJS

const app = express(); // Initialize Express

module.exports = app; // exports a function or object in CommonJS
I det her eksempel, så først importere vi express ved at bruge require(), som er en del af CommonJS modulet. Vi tildeler det en variable som vi kalder for express, det er standard måden at gøre det på, hvis man skal arbejde med express og bruge den variable senere, så er det godt at gemme det i en variable.
Der efter initialiserer vi express, som vi gør ved at kalde på express() funktionen. Hvor vi opretter en ny instans af Express, hvor vi gemmer den i variablen app

Til sidst så eksportere vi Express-appen, som vi gør ved at tildele den module.exports, som gør appen tilgængelig for andre filer, hvis vi arbejde men den, så kan vi importerer denne fil i andre filer.

Hvis man arbejder med ECMAScript-moduler (ES-moduler), som er den nyere standard, som er blevet tilføjet til Node.js. Så skal det gøres anderledes og håndteres anderledes end CommonJS.
I ES-moduler, der bruger man, altså import og export, til at arbejde hvis man skal importere og eksportere moduler. Her er et eksempel. 
import express from "express"; // imports Express in ES-modules

const app = express(); // initialize Express

export default app; // exports a function or objects in ES-modules
Hvis man skal bruge ES-moduler, så skal man også sørge for at man har "type": "module" i ens package.json fil, ellers så kan man også bruge filtypenavne som .mjs til alle sine ES-moduler.
