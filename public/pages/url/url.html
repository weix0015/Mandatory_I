Protocol, det angiver, hvordan information bliver transporteret over internettet. Det kan f.eks. være https og http, men der findes også andre protokoller. Det er en måde, der fortæller din computer, hvordan den skal kommunikere med en anden computer eller server. HTTPS står for HyperText Transfer Protocol Security
HTTP står for HyperText Transfer Protocol
http protocol bliver brugt world wide, til kommunikation af computer til web, hvor https er en mere sikker version af http, som bruges til kommunikation over internettet. HTTPS anvendes til sikker kommunikation, især til følsomme transaktioner som online banking, eller hvis man skal logge ind på hjemmesider og sende data frem og tilbage, så bliver det krypterer dataen, som gør det sværere for angriber at læse og opsnuse dataen.

Domain, det identificerer en specifik hjemmeside eller webserver, som man ønsker at besøge. Det sker f.eks. hver gang bruger sender en GET-request til en server eller website. Men det er den første del i URL og det kan indeholde flere dele. For eksempel, i URL'en https://www.youtube.com/watch?v=jrpWVr18ObQ, så er domain www.youtube.com

Subdomain, det er en del af domain, som er placeret foran hoveddomain. For eksempel, i URL'en www.youtube.com, så er subdomain www.

Domain name, det er navnet og det refererer til den del af domain, som identificerer hjemmesiden eller serveren. For eksempel, i URL'en www.youtube.com, så er domain name youtube.com.

Top level domain (TLD), det er det sidste stykke af et domain eller ., altså punktummet. For eksempel i URL'en www.youtube.com, så er Top level domain .com.

Path, det angiver en specifikt sti, altså hvilken side eller ressource på hjemmesiden du vil have adgang til. Det kan for eksempel være du er på forsiden og du vil gerne hen til siden contact, så vil der stå i URL'en /contact, altså www.map/contact.com, så vil forsiden være map, hvor den side du vil hen til eller er på det er /contact, som er i URL'en efter den første del af domain name. For eksempel på hjemmesiden youtube, så vil det være /watch, hvis du vil hen til en side og se en video. Path, er altid afskilt med en skråstreg /.

Query String Separator, det bruges hvis du skal sende extra information til en server eller hjemmeside. Det starter med et ?, som adskiller det fra den tidligere del af URL'en. For eksempel https://www.example.com/search?q=query&page=1, så vil ? være Query String Separator.

Query String Parameter, det er specifikke oplysninger, man sender til serveren eller hjemmesiden. De er i form af key-value pair. For eksempel med Youtube-videoen er det v som er key og jrpWVr18ObQ, som er value. Det bruges til at identificerer den specifikke video, man vil se eller man ser lige nu. URL'en https://www.youtube.com/watch?v=jrpWVr18ObQ, så vil v være key og jrpWVr18ObQ være value. Så der står i url, nøglens v = værdi jrpWVr18ObQ.
